# Portfolio UI/UX Audit & Fix Plan

## Summary
Comprehensive scan of all components, styles, and interactions. 20+ issues found across accessibility, responsive design, visual bugs, dead code, and interaction gaps.

---

## 1. Critical Visual Bugs

### 1a. HeroScene renders full-page fixed Canvas (always visible)
**File:** `components/three/HeroScene.tsx:132`
- `fixed inset-0` means the 3D scene is ALWAYS behind every section, even when scrolled past hero
- **Fix:** Change to `absolute` positioned inside the hero section only, or add IntersectionObserver to pause/resume

### 1b. Next.js/Express skill icons invisible on light theme
**File:** `components/layout/SkillsSection.tsx:35,42`
- `SiNextdotjs` color `oklch(0.95 0 0)` and `SiExpress` color `oklch(0.65 0 0)` are near-white — invisible against light backgrounds
- **Fix:** Use slightly darker values that work in both themes, or use `currentColor` with a theme-aware approach

### 1c. Hero CTA buttons hardcoded `text-white` breaks light theme
**File:** `components/layout/HeroSection.tsx:144,155`
- `text-white` on primary button is fine, but `FaDownload` has extra `text-secondary` class
- **Fix:** Remove `text-secondary` from FaDownload (it inherits from parent)

### 1d. Project card hover overlay too dark in dark theme
**File:** `components/layout/ProjectsSection.tsx:114`
- `from-background` in dark mode is `oklch(0.07 ...)` — nearly black, making Live/Code buttons invisible
- **Fix:** Use `from-black/60` or `from-background/80` for better contrast

## 2. Responsive / Layout Bugs

### 2a. Education timeline broken on mobile
**File:** `components/layout/EducationSection.tsx:60`
- `left-4 md:left-1/2 -translate-x-1/2` — on mobile, `left: 16px` + `translateX(-50%)` = `left: 8px`, cutting into the content area
- **Fix:** On mobile, use `left-4` without `translate-x`, on md+ use `left-1/2 -translate-x-1/2`

### 2b. Services card flip doesn't work on touch devices
**File:** `components/layout/ServicesSection.tsx:96-97`
- `onMouseEnter`/`onMouseLeave` — no mobile equivalent
- **Fix:** Add `onClick` toggle for touch devices, or use a different interaction pattern

### 2c. Services fixed height `h-[280px]` may clip content on small screens
**File:** `components/layout/ServicesSection.tsx:95`
- Flipped card has 4 list items — could overflow on small viewports
- **Fix:** Use `min-h-[280px]` or make height responsive

### 2d. Skills tooltip clipped by section overflow
**File:** `components/layout/SkillsSection.tsx:51,109`
- Section has `overflow-hidden`, tooltip uses `absolute -bottom-12` — clipped
- **Fix:** Remove `overflow-hidden` from the section, or position tooltip differently

## 3. Accessibility Issues

### 3a. CustomCursor hides cursor on form inputs
**File:** `components/layout/CustomCursor.tsx:18`
- `cursor: none !important` on `input, textarea, select` makes forms unusable
- **Fix:** Remove `input, textarea, select` from the cursor-hide selector

### 3b. No skip-to-content link
**File:** `app/page.tsx`
- **Fix:** Add a visually hidden skip link at the top

### 3c. FunSection modal has no keyboard support
**File:** `components/layout/FunSection.tsx:95-122`
- No Escape key handler, no focus trap
- **Fix:** Add `useEffect` for Escape key and `autoFocus` on close button

### 3d. GitHub sort buttons missing aria-pressed
**File:** `components/layout/GithubProjects.tsx:113-134`
- **Fix:** Add `aria-pressed={sortBy === "updated"}` / `aria-pressed={sortBy === "stars"}`

### 3e. Footer social links missing aria-label
**File:** `components/layout/Footer.tsx:35-56`
- **Fix:** Add `aria-label="GitHub"`, `aria-label="LinkedIn"`, `aria-label="Email"`

### 3f. Contact form inputs missing aria-required
**File:** `components/layout/ContactSection.tsx:136-182`
- **Fix:** Add `required` and `aria-required="true"` to name, email, message fields

## 4. Interaction / UX Gaps

### 4a. Header has no active section indicator
**File:** `components/layout/Header.tsx`
- No visual cue for which section is in viewport
- **Fix:** Add ScrollTrigger-based active nav highlighting

### 4b. Mobile menu has no animation
**File:** `components/layout/Header.tsx:110-125`
- Menu appears/disappears instantly
- **Fix:** Add slide-down animation with GSAP or CSS transition

### 4c. No ScrollTrigger.refresh() after GitHub data loads
**File:** `lib/useGsap.ts` + `components/layout/GithubProjects.tsx`
- Dynamic content (GitHub cards) loads after GSAP initializes — ScrollTrigger positions are stale
- **Fix:** Call `ScrollTrigger.refresh()` after repos load

### 4d. Project card 3D tilt conflicts with GSAP animations
**File:** `components/layout/ProjectsSection.tsx:76`
- Inline `transform` from tilt effect overwrites GSAP's transform
- **Fix:** Use separate wrapper for tilt, or apply tilt to inner element only

## 5. Dead Code Cleanup

### 5a. Unused CSS classes in globals.css
**File:** `app/globals.css:244-274`
- `.animate-on-scroll`, `.visible`, `.glass`, `.hover-lift` — defined but never used
- **Fix:** Remove dead CSS

### 5b. Unused import in FunSection
**File:** `components/layout/FunSection.tsx:5`
- `Image` imported but never used
- **Fix:** Remove import

### 5c. Unused keyframe animations
**File:** `app/globals.css:185-208`
- `fade-in-up`, `slide-in-left`, `slide-in-right`, `scale-in`, `letter-reveal` — their utility classes exist but GSAP handles these now
- **Fix:** Remove unused keyframes and utility classes

---

## 6. Performance

### 6a. HeroScene Canvas always rendering
**File:** `components/three/HeroScene.tsx:130-145`
- Fixed position Canvas renders 3D even when scrolled to footer
- **Fix:** Use `frameloop="demand"` or IntersectionObserver to pause when hero is offscreen

### 6b. CustomCursor re-renders on every mousemove
**File:** `components/layout/CustomCursor.tsx:21`
- `setPos` triggers React re-render 60fps
- **Fix:** Use refs for position, update DOM directly
